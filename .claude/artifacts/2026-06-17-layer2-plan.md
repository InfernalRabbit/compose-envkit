## 0. The minimal levers (core finding)

Contract: *nothing a service `env_file:` defines reaches `COMPOSE_ENV_FILES`, but
`ProjectView.Services` enumeration survives for debug.* Met by TWO levers:

- **Run path:** `cmd/cenvkit/main.go:100` `assemble()` calls
  `envfiles.Assemble(cr.Files, er.EnvFiles)`. Change the Layer-2 arg to `nil`
  (and per D1, drop the now-dead `engine.Resolve` call from the run path).
- **Debug path:** `internal/engine/provenance.go:80-97` folds ALL `in.EnvFiles`
  (both layers) into `chainEnv`, which seeds the B-lite `mapping` AND the C-load.
  Add an `interpEnv` (Layer-1 + shell only) and base the B-lite `mapping` on it;
  `chainEnv` (all-layers) stays for A-attribution overlay and the C `mergedEnv`.

Everything else (gap fields, gap flagging, renderer) builds on these two.

## 2. `provenance.Report` model delta (spec §3.3)

```go
type ServiceVal struct { // NEW — one service env_file def of a var (gap evidence; runtime-only)
	Service string `json:"service"`
	File    string `json:"file"`
	Value   string `json:"value"`
}
type Effect struct {
	Service  string `json:"service"`
	Field    string `json:"field"`
	Resolved string `json:"resolved"` // now the REAL run value (fallback when env_file-only)
	Gap      bool   `json:"gap"`      // NEW
}
type VarTrace struct {
	Name        string       `json:"name"`
	Value       string       `json:"value"`      // attribution over the INTERPOLATION env only (L1 + shell)
	Winner      Source       `json:"winner"`     // "" winner when env_file-only
	Overridden  []Source     `json:"overridden,omitempty"`
	Effects     []Effect     `json:"effects,omitempty"`
	InChain     bool         `json:"in_chain"`            // NEW
	RuntimeDefs []ServiceVal `json:"runtime_defs,omitempty"` // NEW
	Gap         bool         `json:"gap"`                 // NEW = referenced && !InChain && len(RuntimeDefs)>0
}
```

L1-only A-attribution falls out of T4+T7 (no extra code): once `Report.Files` is
L1-only and cmd passes only L1 ProvFiles, A attributes over L1+shell naturally.

## 3. L1-only mapping — before/after (`provenance.go:77-97`)

Add `interpEnv` (layer1 files + shell `in.Env`); base `mapping` on it:

```go
chainEnv := map[string]string{}   // all layers — A overlay + C load (unchanged role)
interpEnv := map[string]string{}  // L1 + shell ONLY — the REAL run interpolation env
parsed := make([]map[string]string, len(in.EnvFiles))
lookup := dotenv.LookupFn(func(k string) (string, bool) { v, ok := chainEnv[k]; return v, ok })
for i, f := range in.EnvFiles {
	if m, err := parseDotEnv(f.Path, lookup); err == nil {
		parsed[i] = m
		for k, v := range m {
			chainEnv[k] = v
			if f.Layer == "layer1" { interpEnv[k] = v }
		}
	}
}
for k, v := range envSliceToMap(in.Env) { chainEnv[k] = v; interpEnv[k] = v }
mapping := template.Mapping(func(k string) (string, bool) { v, ok := interpEnv[k]; return v, ok })
```

Keep the explicit `f.Layer=="layer1"` filter even though after T7 `in.EnvFiles`
is L1-only at runtime — it makes the engine correct regardless of caller
(defensive, crisp unit contract).

Effects → real run value: B-lite `template.SubstituteWithOptions(leaf, mapping,
WithoutLogging)` with `mapping` reading `interpEnv` (no `WEB_PORT`) → compose-go
`withDefaultWhenAbsence` (`template/template.go:323`, VERIFIED) takes the `:-`
branch → `"${WEB_PORT:-0}:80" → "0:80"`.

**CORRECTION (D5, supersedes the original "keep chainEnv for C" line):** the C
resolved-load's `cli.WithEnv` MUST be fed `interpEnv` (Layer-1 + shell only), NOT
`chainEnv`. At the real run inline `environment:` `${VAR}` interpolates against
`COMPOSE_ENV_FILES` (= L1) + host — never a service `env_file:` — so feeding
chainEnv would make `--effective` show an env_file-only `${X}` resolving when the
container actually gets the fallback (the debugger would lie). env_file *literal*
entries (`dotenv.ReadFile(svc.EnvFiles)`) are injected verbatim and are unaffected;
inline-`environment:`-overrides-env_file precedence is unchanged. Net: only the C
load's WithEnv source flips chainEnv → interpEnv (+ the raw B-lite load's
`details.Environment` → interpEnv for consistency; cosmetic since SkipInterpolation).
See spec §3 point 2 + §8 D5.

## 4. Gap detection (`provenance.go`, after C-loop + B-lite walk; pure Go)

Build `runtimeDefs` inside the existing C-loop (reuses its `dotenv.ReadFile`;
`proj.Services` is the active/profile-filtered set):
```go
runtimeDefs[k] = append(runtimeDefs[k], provenance.ServiceVal{Service: name, File: ef.Path, Value: v})
```
Then:
```go
for name, vt := range rep.Vars {
	_, vt.InChain = interpEnv[name]
	vt.RuntimeDefs = runtimeDefs[name]
	referenced := len(vt.Effects) > 0
	vt.Gap = referenced && !vt.InChain && len(vt.RuntimeDefs) > 0
	for i := range vt.Effects { vt.Effects[i].Gap = vt.Gap }
	rep.Vars[name] = vt
}
```
- `referenced` = `len(Effects)>0` (Effects exist only for service fields — B-lite
  scope, `splitServiceField` provenance.go:297). Non-service-field refs are not
  gap-candidates (consistent w/ §3.1 / B-lite non-goal).
- unset-everywhere → `RuntimeDefs` empty → `Gap=false` (acceptance #4).
- in-chain → `InChain=true` → `Gap=false`, normal winner (acceptance #4).
- Sort each `RuntimeDefs` by `(Service,File)` (match the Effects sort 243-251) for
  stable `--json`.

`--trace` render (`render.go`): in-chain → normal winner/overridden/effects, no
gap line. Not-in-chain → `interpolation: NOT in the Layer-1 chain …` + a
`runtime:` line per `RuntimeDefs` + (if Gap) a `⚠ gap:` line per Effect citing the
field + resolved fallback + a `fix:` line. Exit 0 (warning, not error — §3.5).
